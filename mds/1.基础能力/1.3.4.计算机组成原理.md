## 定点数

| 维度 | 内容 |
|------|------|
| **作用** | 固定小数点，性能比浮点高 |
| **应用场景** | 跨平台一致性；低配优化；确定精度 |
| **类型** | 定点小数（有符号/无符号）；定点整数 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **溢出** | 注意范围；更大类型；检查；饱和运算 |
| **精度限制** | 合适格式；高精度；关键处用浮点 |
| **实现复杂** | 定点数库；封装函数；宏/内联 |

## 优点

| 优点 | 说明 |
|------|------|
| 性能更高 | 比浮点快 |
| 精度确定 | 不因平台而异 |
| 跨平台一致 | 运算结果一致 |

## 应用举例

| 场景 | 说明 |
|------|------|
| 跨平台一致性 | 不同设备浮点精度不同，同步用定点数 |
| 低配优化 | 掌机等低配平台 |

**要点和思考方向**
- 定点数适合需要确定精度和跨平台一致性的场景
- 性能比浮点数高，但实现相对复杂
- 注意溢出和精度限制
- 游戏开发中常用于物理计算、位置同步等

## 浮点数

| 维度 | 内容 |
|------|------|
| **作用** | 科学计数法表示实数 |
| **应用场景** | 常规计算；物理；图形；大/小数值 |
| **类型** | float（32 位，约 7 位）；double（64 位，约 15 位） |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **精度丢失** | double；定点数；避免累积；epsilon 比较 |
| **跨平台精度差异** | 定点数；明确要求；测试；标准化格式 |
| **性能问题** | 定点数；SIMD；减少运算；快速数学函数 |

## 优点

| 优点 | 说明 |
|------|------|
| 使用灵活 | 可表示大/小数值 |
| 精度足够 | 大多数应用足够 |
| 实现简单 | 直接使用 |

**要点和思考方向**
- 浮点数适合大多数数值计算场景
- 注意精度丢失和跨平台精度差异
- 游戏开发中通常使用float（单精度）以节省内存
- 在关键位置（如物理计算）可以使用double（双精度）

## 常见类型

| 位数 | 类型 | 范围 |
|------|------|------|
| 8 | char、byte | -128~127 或 0~255 |
| 16 | short | -32768~32767 |
| 32 | int、float | ±21 亿或 ±3.4×10³⁸ |
| 64 | long、double | 更大 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **溢出** | 更大类型；检查；无符号；饱和运算 |
| **内存浪费** | 按需求选择；内存紧张时优化 |
| **精度不足** | 高精度类型；定点/浮点；重设计结构 |

| 要点和思考方向 |
|----------------|
| 移动平台注意内存 |
