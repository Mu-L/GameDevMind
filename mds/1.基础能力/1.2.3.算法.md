## 复杂度

## 时间复杂度

| 维度 | 内容 |
|------|------|
| **作用** | 衡量算法执行时间随输入规模增长的变化趋势 |
| **应用场景** | 算法设计与选择；性能评估与优化；系统设计权衡 |
| **应用** | 根据需求权衡时间换空间或空间换时间 |

## 应用举例

| 场景 | 方案 | 权衡 |
|------|------|------|
| 预计算 vs 实时计算 | 有限数列且性能要求高时，预存到数组 | 用空间换时间 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **复杂度分析不准确** | 考虑常数因子；实际测试；分析最坏/平均/最好；考虑数据特征 |
| **空间复杂度忽略** | 同时分析时空；按资源限制选算法；考虑内存受限 |

| 要点和思考方向 |
|----------------|
| 复杂度帮助选算法；时空权衡；理论需实际验证；注意常数因子 |

## 排序

| 维度 | 内容 |
|------|------|
| **作用** | 将数据按规则重新排列 |
| **应用场景** | 数据展示（排行榜）；优化查找；预处理；算法辅助 |

## 方法

| 算法 | 特点 | 复杂度 |
|------|------|--------|
| 冒泡排序 | 简单，效率低 | O(n²) |
| 选择排序 | 每次选最小/最大 | O(n²) |
| 插入排序 | 适合部分有序 | O(n²)，最好 O(n) |
| 快速排序 | 平均性能好，广泛使用 | 平均 O(n log n)，最坏 O(n²) |
| 归并排序 | 稳定，适合大规模 | O(n log n) |
| 堆排序 | 使用堆结构 | O(n log n) |
| 计数排序 | 适合小范围整数 | O(n+k) |

## 应用举例

| 场景 | 需求/方案 |
|------|-----------|
| 游戏排行榜 | 对玩家分数排序，常用快排、归并 |
| 碰撞检测 | 按位置排序后只检测相邻物体 |
| 搜索算法 | 启发式搜索需对队列排序 |
| AI 决策 | 按评估值排序后选最高决策 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **算法选择不当** | 按规模选；小规模用插入；大规模用快排/归并；考虑部分有序 |
| **稳定性要求** | 归并/插入；比较函数加原始位置；稳定实现 |
| **性能问题** | 高效算法；增量排序；并行；标准库 |

| 要点和思考方向 |
|----------------|
| 按规模和特征选算法；注意稳定性；小规模可能简单算法更快 |

## 查找

| 维度 | 内容 |
|------|------|
| **作用** | 在数据集合中定位特定元素 |
| **应用场景** | 数据检索；玩家匹配；资源查找；缓存查找 |

## 方法

| 方法 | 特点 | 复杂度 |
|------|------|--------|
| 二分查找 | 有序数组 | O(log n) |
| 散列表查找 | 哈希表 | 平均 O(1) |
| 树结构查找 | 二叉搜索树等 | O(log n) |
| 线性查找 | 顺序遍历 | O(n) |

## 应用举例

| 场景 | 说明 |
|------|------|
| 玩家匹配 | 快速查找匹配玩家 |
| 资源管理 | 查找游戏资源 |
| 缓存系统 | 查找缓存 |
| 配置查找 | 查找配置项 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **查找效率低** | 二分（有序）；哈希表；树结构；建立索引 |
| **数据需排序** | 哈希表；自平衡树；增量排序 |
| **内存开销** | 小规模用线性；紧凑结构；权衡内存与性能 |

| 要点和思考方向 |
|----------------|
| 按数据特征选方法；有序用二分；考虑维护成本 |

## 搜索

| 维度 | 内容 |
|------|------|
| **作用** | 在树或图中查找目标节点或路径 |
| **应用场景** | 寻路；状态空间搜索；树/图遍历；问题求解 |

## 深度优先搜索（DFS）

| 维度 | 内容 |
|------|------|
| **作用** | 沿深度遍历，尽可能深地搜索分支 |
| **应用场景** | 探索所有路径；回溯；拓扑排序；连通性检测 |
| **特点** | 栈（递归）实现；可能非最短路径 |

| 问题 | 解决方向 |
|------|----------|
| **栈溢出** | 迭代实现；限制深度；显式栈 |
| **非最短路径** | 需最短路径用 BFS；深度限制；IDS |

| 要点和思考方向 |
|----------------|
| 适合探索所有路径；注意栈溢出；适合回溯 |

## 广度优先搜索（BFS）

| 维度 | 内容 |
|------|------|
| **作用** | 沿宽度遍历树的节点 |
| **应用场景** | 最短路径；层次遍历；连通性检测 |
| **特点** | 队列实现；能找到最短路径 |

| 问题 | 解决方向 |
|------|----------|
| **内存开销** | 双向 BFS；限制范围；IDBFS |
| **性能问题** | A*；并行 BFS；优化队列 |

| 要点和思考方向 |
|----------------|
| 适合最短路径；注意内存；适合无权图 |

## 应用举例

| 场景 | DFS | BFS |
|------|-----|-----|
| 寻路 | 探索所有路径 | 寻找最短路径 |
| 状态空间搜索 | 搜索游戏状态空间 | - |
| 连通性检测 | 检测图连通性 | - |

## 填充算法（Hole-Filling）

| 维度 | 内容 |
|------|------|
| **作用** | 从内点开始，由内向外填充至边界 |
| **应用场景** | 图像处理；游戏机制；地图生成；区域标记 |

## 算法

| 算法 | 说明 |
|------|------|
| Lazy flood fill | 延迟填充，优化性能 |
| 洪水填充 | 从起点向四周扩散 |
| 边界填充 | 沿边界填充 |

## 应用举例

| 场景 | 说明 |
|------|------|
| 消除游戏 | 消除同色区域 |
| 寻路 | 标记可达区域 |
| 地图生成 | 生成连续地形 |
| 画笔功能 | 填充工具 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **性能问题** | Lazy flood fill；限制范围；扫描线；分块 |
| **栈溢出** | 迭代实现；队列替代递归；限制深度 |
| **边界检测** | 精确算法；预处理；抗锯齿 |

| 要点和思考方向 |
|----------------|
| 注意性能；迭代避免栈溢出；注意边界检测 |

| 维度 | 内容 |

| **应用场景** | 树/图遍历；分治；回溯；自然递归结构 |
| **特点** | 代码简洁；可能栈溢出、性能开销大 |

## 应用举例

| 场景 | 说明 |
|------|------|
| 遍历文件目录 | 递归遍历目录树 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **栈溢出** | 迭代替代；限制深度；增加栈空间；尾递归优化 |
| **性能开销** | 迭代；尾递归；记忆化 |

| 要点和思考方向 |
|----------------|
| 注意栈溢出；深度大用迭代；尾递归减少开销 |

## 模式

## 分治法

| 维度 | 内容 |
|------|------|
| **作用** | 分解为子问题，递归解决后合并 |
| **应用场景** | 可分解为相似子问题；解可合并；适合并行 |
| **应用** | 快排、归并、大整数乘法 |

| 问题 | 解决方向 |
|------|----------|
| **子问题重叠** | 动态规划；缓存子问题；合并相似子问题 |
| **合并复杂度** | 优化合并算法；简化逻辑；并行合并 |

| 要点和思考方向 |
|----------------|
| 注意子问题重叠；优化合并过程 |

## 动态规划法

| 维度 | 内容 |
|------|------|
| **作用** | 保存子问题解避免重复计算 |
| **应用场景** | 最优子结构；子问题重叠；需最优解 |
| **应用** | 最短路径；背包；最长公共子序列 |

| 问题 | 解决方向 |
|------|----------|
| **状态定义困难** | 从简单情况分析；表格记录；参考经典 |
| **空间复杂度高** | 滚动数组；只保存必要状态；记忆化搜索 |
| **边界条件** | 仔细定义初始状态；边界检查；测试验证 |

| 要点和思考方向 |
|----------------|
| 注意空间优化；仔细定义状态转移；注意边界 |

## 贪心算法

| 维度 | 内容 |
|------|------|
| **作用** | 每步选当前最优，期望全局最优 |
| **应用场景** | 贪心选择性质；局部最优→全局最优；快速近似解 |
| **应用** | 最小生成树；最短路径；活动选择 |
| **注意** | 不一定全局最优；需证明正确性 |

| 问题 | 解决方向 |
|------|----------|
| **非全局最优** | 证明贪心性质；DP 验证；接受近似解 |
| **策略选择** | 分析问题性质；尝试不同策略；参考经典 |

| 要点和思考方向 |
|----------------|
| 需证明正确性；不一定全局最优；适合快速近似 |
