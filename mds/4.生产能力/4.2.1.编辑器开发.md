<h2 align="center">编辑器开发</h2>
<p>
虽然现在越来越多的工具都已经由引擎或者第三方实现了，但很多团队仍然需要为自己产品定制编辑器和小工具，以适配特定的数据结构、工作流和协作方式。本章节聚焦「怎么设计一个好用的游戏编辑器」的原则和关键功能点。
</p>

*所见即所得,Undo/Redo,视图,面板,工程文件,插件系统*

## 设计原则

## 所见即所得

| 维度 | 内容 |
|------|------|
| **作用** | 让策划、美术在编辑器中看到的效果，与游戏中真实效果尽可能一致，减少反复调试和来回切换 |
| **应用场景** | 游戏内容制作、美术资源制作、关卡设计、特效制作 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现所见即所得** 需要让编辑器预览效果与游戏运行时一致 | 预览视图使用与运行时一致的渲染管线与资源；关键参数修改后能实时反馈；必要时提供「运行时模拟」功能（播放特效、物理模拟）；使用相同的渲染系统；实现实时预览机制 |

| 要点与思考方向 | 说明 |
|----------------|------|
| **渲染一致性** | 使用相同的渲染管线、资源格式；实现实时预览 |
| **实时反馈** | 参数修改后能实时反馈；实现参数实时更新、预览刷新 |
| **运行时模拟** | 播放特效、物理模拟、动画播放；提供完整预览 |

| 要点和思考方向 |
|----------------|
| 所见即所得是编辑器设计的核心原则 |
| 使用与运行时一致的渲染管线保证效果一致 |
| 实现实时反馈提高编辑效率 |
| 提供运行时模拟功能增强预览能力 |
