## 1.1 编程语言 — 你的第一把武器

### 是什么？在哪用？

编程语言是开发者与计算机沟通的工具。在游戏开发中，不同的岗位和平台会用到不同的语言：

| 语言 | 主要用途 | 代表引擎/框架 |
|------|----------|---------------|
| **C++** | 游戏引擎开发、高性能模块、主机游戏 | Unreal、自研引擎 |
| **C#** | 游戏逻辑开发、Unity 游戏 | Unity、MonoGame |
| **JavaScript/TypeScript** | Web 游戏、小游戏、H5 | Cocos、Egret、Phaser |
| **Lua/Python** | 脚本系统、热更新、工具开发 | 各类引擎的脚本层 |

### 会遇到哪些问题？用什么解决？

| 问题 | 解决方向 |
|------|----------|
| 内存泄漏 | 学习智能指针、RAII、垃圾回收机制 |
| 性能瓶颈 | 了解语言特性、避免不必要的分配、使用对象池 |
| 类型安全 | 使用静态类型语言或 TypeScript 等强类型语言 |
| 跨平台兼容 | 注意语言在不同平台的差异，使用条件编译 |

### 💡 要点和思考方向

1. **不要只学语法**：理解语言背后的设计思想和适用场景
2. **至少精通一门**：深入理解一门语言，再学其他语言会更容易
3. **了解底层原理**：知道代码最终如何变成机器指令
4. **关注内存管理**：游戏对性能敏感，内存管理是基本功

## 1.2 程序设计 — 写出好代码的艺术

### 1.2.1 设计模式

**是什么？在哪用？**

设计模式是解决常见设计问题的**可复用方案**。在游戏开发中，设计模式无处不在：

| 模式 | 游戏开发中的应用场景 |
|------|----------------------|
| **单例模式** | 游戏管理器、资源管理器 |
| **工厂模式** | 创建游戏对象、生成敌人 |
| **观察者模式** | 事件系统、成就系统 |
| **状态模式** | 角色状态机、游戏状态管理 |
| **策略模式** | AI 行为、技能系统 |
| **对象池模式** | 子弹、粒子效果等频繁创建销毁的对象 |
| **组合模式** | 场景树、UI 层级 |

### 会遇到哪些问题？用什么解决？

| 问题 | 解决方向 |
|------|----------|
| 过度设计 | 先实现功能，再考虑重构，不要为了模式而模式 |
| 模式滥用 | 理解模式的本质，而不是死记硬背 |
| 难以维护 | 遵循 SOLID 原则，保持代码简洁 |

### 💡 要点和思考方向

1. **模式是工具，不是目的**：用合适的模式解决合适的问题
2. **理解比记忆重要**：理解模式解决的问题，而不是死记结构
3. **在实践中学习**：通过重构来体会模式的价值

**游戏开发中常用的数据结构：**

| 数据结构 | 游戏应用场景 |
|----------|--------------|
| **数组/列表** | 角色列表、物品栏 |
| **字典/哈希表** | 资源缓存、配置表 |

## 1.3 通用基础 — 地基越深，楼越高

### 1.3.1 数学

**游戏开发中必需的数学知识：**

| 数学领域 | 游戏应用场景 |
|----------|--------------|
| **线性代数** | 向量运算、矩阵变换、3D 图形 |
| **三角函数** | 旋转、波动、周期性运动 |
| **几何** | 碰撞检测、射线检测 |
| **概率统计** | 随机事件、掉落率、战斗公式 |
| **微积分** | 物理模拟、运动轨迹 |

### 💡 学习建议

- **不必成为数学家**：理解概念和应用场景即可
- **结合实践学习**：通过实现具体功能来理解数学
- **善用工具**：Unity 的 Vector3、Quaternion 等已经封装好了常用运算

---

### 1.3.2 操作系统

**需要了解的概念：**

- 进程与线程（多线程编程、任务系统）
- 内存管理（虚拟内存、分页）
- 文件系统（资源加载、存档）
- I/O 操作（异步加载、流式读取）

---

### 1.3.3 计算机网络

**游戏开发中的网络知识：**

| 概念 | 游戏应用场景 |
|------|--------------|
| **TCP/UDP** | TCP 用于可靠传输（登录、支付），UDP 用于实时对战 |
| **HTTP/HTTPS** | API 调用、资源下载 |
| **WebSocket** | 实时通信、推送通知 |
| **Socket 编程** | 自定义网络协议 |
| **网络同步** | 状态同步、帧同步 |

## 🎯 学习路径建议

### 对于新人

```
1. 先精通一门语言（推荐 C# 或 C++）
   ↓
2. 学习基础数据结构和算法
   ↓
3. 了解常用设计模式
   ↓
4. 补充数学和计算机基础
   ↓
5. 通过实战项目巩固
```

### 对于在职开发者

- **查漏补缺**：对照本章目录，找到自己的知识盲区
- **按需学习**：遇到问题时，针对性地补充相关知识
- **深度优先**：先深入理解工作中用到的部分，再扩展

## 🤖 AI Coding 指南

### 如何让 AI 帮你学习基础能力？

| 任务类型 | 示例提示词 |
|----------|------------|
| **概念解释** | "用游戏开发的例子解释什么是观察者模式" |
| **代码示例** | "用 C# 写一个对象池的示例，用于管理子弹" |

| **重构建议** | "这段代码违反了哪些设计原则，如何重构" |
| **算法实现** | "用 A* 算法实现一个寻路功能，带注释" |

### 注意事项

1. **提供上下文**：告诉 AI 你的使用场景（Unity/Unreal/自研引擎）
2. **指定语言**：明确说明使用的编程语言
3. **验证输出**：AI 生成的代码需要自己理解和测试

## ✍️ 本章小结

基础能力是游戏开发的**地基**。地基越深，楼才能盖得越高。

**核心要点：**

1. **至少精通一门编程语言**，理解其特性和底层原理
2. **掌握常用设计模式**，但不要过度设计
3. **理解数据结构和算法**，这是写出高效代码的基础
4. **补充数学和计算机基础**，这些知识会在关键时刻派上用场
