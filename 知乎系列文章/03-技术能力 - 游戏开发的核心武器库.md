## 2.1 客户端技术 — 让玩家看到、听到、感受到

### 2.1.1 图形渲染

**是什么？在哪用？**

图形渲染是将 3D/2D 数据转换为屏幕上的像素的过程。这是游戏最核心的技术之一。

**核心概念：**

| 概念 | 说明 |
|------|------|
| **渲染管线** | 顶点处理 → 几何处理 → 光栅化 → 像素处理 |
| **Shader** | 运行在 GPU 上的小程序，控制渲染效果 |
| **材质** | 定义物体表面如何与光线交互 |
| **光照** | 模拟真实世界的光线行为 |
| **后处理** | 渲染完成后的图像处理（模糊、景深、色调映射） |

**常见问题与解决：**

| 问题 | 解决方向 |
|------|----------|
| 帧率低 | LOD、遮挡剔除、批处理、减少 DrawCall |
| 画面锯齿 | MSAA、FXAA、TAA 等抗锯齿技术 |
| 内存占用高 | 纹理压缩、资源流式加载 |

---

### 2.1.2 物理系统

**是什么？在哪用？**

物理系统模拟真实世界的物理规律，让游戏世界更加真实。

**核心功能：**

- **碰撞检测**：判断物体是否相交
- **刚体模拟**：模拟物体的运动和碰撞
- **柔体/布料模拟**：模拟柔软物体的形变
- **流体模拟**：模拟水、烟等流体效果
- **车辆物理**：模拟车辆的运动和操控

**常用引擎：**

- Unity：内置 PhysX
- Unreal：内置 Chaos/PhysX
- 自研：Bullet、Box2D（2D）

---

### 2.1.3 UI 系统

**是什么？在哪用？**

UI 系统是玩家与游戏交互的界面，包括菜单、HUD、对话框等。

**核心概念：**

| 概念 | 说明 |
|------|------|
| **锚点** | UI 元素相对于父容器的定位方式 |
| **自适应** | 适配不同分辨率和屏幕比例 |
| **事件系统** | 处理点击、拖拽等交互 |
| **UI 合批** | 合并 UI 绘制，提升性能 |

**常见问题：**

- UI 闪烁：检查锚点设置、避免频繁重建
- 性能问题：减少 UI 元素数量、使用图集
- 适配问题：使用锚点和相对布局

---

### 2.1.4 游戏引擎

**主流引擎对比：**

| 引擎 | 语言 | 优势 | 适用场景 |
|------|------|------|----------|
| **Unity** | C# | 易上手、生态丰富、跨平台 | 手游、独立游戏、VR/AR |
| **Unreal** | C++/蓝图 | 画面效果好、功能强大 | 3A 大作、主机游戏 |
| **Cocos** | JS/TS | 轻量、适合 2D | 2D 手游、小游戏 |
| **Godot** | GDScript | 开源、轻量 | 独立游戏、2D 游戏 |

**选择建议：**

- **新手入门**：Unity（资料多、社区活跃）
- **追求画质**：Unreal（渲染效果好）
- **2D 游戏**：Unity/Cocos
- **小游戏**：Cocos/LayaAir

## 2.2 服务端技术 — 支撑游戏的幕后英雄

### 2.2.1 网络通信

**核心概念：**

| 概念 | 说明 |
|------|------|
| **TCP** | 可靠传输，用于登录、支付等 |
| **UDP** | 不可靠但快速，用于实时对战 |
| **HTTP/HTTPS** | RESTful API、资源下载 |
| **WebSocket** | 长连接、实时推送 |
| **Protobuf** | 高效的数据序列化协议 |

**网络同步方案：**

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| **状态同步** | 服务器同步游戏状态 | MMO、SLG |
| **帧同步** | 同步操作指令，客户端模拟 | MOBA、RTS |
| **混合同步** | 结合两者优点 | 复杂游戏 |

---

### 2.2.2 数据库

**游戏开发中常用的数据库：**

| 数据库 | 类型 | 游戏应用场景 |
|--------|------|--------------|
| **MySQL** | 关系型 | 用户数据、订单、配置表 |
| **Redis** | 键值对 | 缓存、排行榜、会话 |
| **MongoDB** | 文档型 | 日志、非结构化数据 |
| **SQLite** | 嵌入式 | 本地存档、小游戏数据 |

---

### 2.2.3 安全防护

**常见攻击与防护：**

| 攻击类型 | 防护措施 |
|----------|----------|
| **DDoS** | CDN、流量清洗、限流 |
| **SQL 注入** | 参数化查询、输入验证 |
| **外挂/作弊** | 服务端验证、反作弊 SDK |
| **数据篡改** | 数据签名、加密传输 |

## 🎯 学习建议

### 客户端开发者

1. **精通一个引擎**：Unity 或 Unreal，深入理解其架构
2. **理解渲染基础**：不必成为图形专家，但要懂基本原理
3. **掌握性能优化**：Profiler 的使用、常见优化手段
4. **了解网络基础**：即使是客户端，也要懂网络通信

### 服务端开发者

1. **掌握网络编程**：Socket、协议设计、并发处理
2. **熟悉数据库**：SQL、索引优化、事务处理
3. **了解游戏业务**：理解游戏逻辑，才能设计好架构
4. **重视安全**：安全防护是服务端的核心职责
